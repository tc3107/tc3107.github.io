Case Study

Linux Server Hardening Lab

A repeatable baseline for reducing common Linux server risks while keeping administration practical and well documented.

Goal

Create a hardening checklist that can be applied to a fresh Linux server and reviewed during routine maintenance.

Hardening Areas

  • SSH configuration with key-based access and reduced password exposure.
  • Firewall rules that expose only required services.
  • Package update checks and restart planning.
  • Least-privilege user access and sudo review.
  • Basic log review for authentication and service events.

Implementation

  • Prepared an initial access checklist before changing SSH settings.
  • Configured UFW rules and confirmed listening services with ss.
  • Reviewed users, groups, sudo access, and file permissions for sensitive paths.
  • Added fail2ban-style protection concepts for repeated authentication failures.
  • Documented rollback steps for firewall and SSH changes to reduce lockout risk.

What This Demonstrates

The project demonstrates security fundamentals, operational caution, documentation discipline, and the ability to translate baseline security controls into practical Linux administration tasks.

Skills Used
