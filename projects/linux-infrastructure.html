Case Study

Small Business Technology Environment

A sanitized overview of my current work supporting a live business environment across server hardware, Linux services, networking, connected PCs, employee support, monitoring, and day-to-day systems administration.

Context

I currently handle IT systems administration for a business environment. The environment needs to stay usable for day-to-day work, which means hardware, storage, local networking, endpoints, remote access, backups, and user support all have to function together reliably.

Problem / Goal

The goal is to maintain a practical, dependable environment without a large internal IT team: keep systems available, keep users supported, keep data recoverable, and keep changes manageable and documented.

My Role

I own, configure, and maintain the main server hardware. I am also responsible for the surrounding systems: Linux services, company domain setup, local networking, connected Windows and Linux PCs, employee support, monitoring, backups, and day-to-day troubleshooting.

What I Built or Maintained

  • Linux server setup, service administration, updates, log review, and operational troubleshooting.
  • Local networking, endpoint connectivity, and practical support for Windows and Linux workstations.
  • Domain-related administration and the systems needed for employees to access shared business resources.
  • Monitoring, alerts, and routine operational checks to catch issues before they become outages.
  • Local and remote user support, including clear communication when troubleshooting with non-technical staff.

Tools / Technologies

Linux, Windows, SSH, networking fundamentals, DNS/domain administration, remote support workflows, system logs, monitoring, and internal documentation.

Security / Reliability Considerations

This case study is intentionally sanitized and excludes internal topology, private IPs, credentials, exact firewall rules, and sensitive business details. In practice, the work includes access control, updates, service hygiene, backup awareness, monitoring, and cautious change management so reliability and maintainability improve together.

What This Demonstrates to Employers

I can operate as a practical technical generalist in a live environment: working across hardware, Linux, networking, support, documentation, troubleshooting, and day-to-day operations rather than staying inside a single narrow role.

Skills Demonstrated

Server hardware Linux administration Windows support Networking Remote support Monitoring Troubleshooting Documentation Systems operations
