Case Study

Small Business Infrastructure Administration

A sanitized overview of my current work administering the IT environment for a small accounting firm, including hardware, Linux services, local networking, employee support, backups, monitoring, and security controls.

Context

The business needed reliable systems for day-to-day accounting work, internal file access, backups, workstation support, remote administration, and secure operations. I became the person responsible for designing, setting up, maintaining, and troubleshooting the environment.

My Role

  • Purchased, own, configured, and maintain the main server hardware, a Dell PowerEdge R720.
  • Set up the server environment, local networking, company domain, and connected Windows/Linux computers.
  • Maintain Linux services used for business workflows, file access, synchronisation, and automation.
  • Provide ongoing IT support to employees both locally and remotely.
  • Manage system security, updates, access controls, firewalling, monitoring, and alerting.
  • Write custom scripts and software to support internal company workflows.

Systems & Responsibilities

  • Server administration: Linux installation, service configuration, SSH administration, updates, logs, uptime checks, troubleshooting, and performance awareness.
  • Networking: Local network setup, firewall rules, DNS/domain management, remote access considerations, and connectivity troubleshooting.
  • Workstations: Support for Windows and Linux PCs used by employees, including software issues, access problems, and general technical troubleshooting.
  • Operations: Monitoring and alerting using a mix of existing tools and custom software where needed.
  • Security: Practical hardening, least-privilege thinking, firewall control, update hygiene, access review, backup planning, and data protection awareness.

How This Maps to Employer Needs

This work is directly relevant to IT support, help desk, junior systems administration, NOC, SOC, and cybersecurity-adjacent roles because it combines user support with real infrastructure ownership. I am not only learning tools in a lab; I am responsible for keeping a real environment usable, recoverable, and secure.

What I Learned

  • Reliability depends on boring fundamentals: backups, documentation, monitoring, access control, and clear escalation paths.
  • Support work is not just fixing issues; it is communicating clearly with non-technical users and preventing repeat problems.
  • Security must fit the business. Controls need to reduce risk without blocking normal work.
  • Owning a system end-to-end teaches how hardware, Linux, networking, cloud services, users, and software interact.
