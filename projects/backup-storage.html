Case Study

Storage, Synchronisation & Backups

A practical storage and recovery-focused case study covering Nextcloud, rsync, NAS-style workflows, permissions, file access, and backup discipline in a live business environment.

Context

Business data has to be accessible, organised, recoverable, and understandable to the people using it. In a small environment, file access, synchronisation, storage growth, and backups quickly become operational concerns rather than background tasks.

Problem / Goal

The goal is to give staff dependable file access while preserving permissions, maintaining recoverability, and avoiding fragile one-copy workflows. Storage needs to stay usable day to day and defensible when something goes wrong.

My Role

I maintain storage and sync systems using Nextcloud, rsync, NAS-style workflows, and Backblaze-backed backups. That includes practical decisions around file access, permissions, backup routines, and recovery-minded administration.

What I Built or Maintained

  • Business file access and synchronisation workflows for everyday use.
  • Rsync-based data movement and backup routines to reduce dependence on a single copy or a single device.
  • NAS-style operational patterns for shared storage, organisation, and permissions.
  • Cloud-backed backup coverage using Backblaze as part of recovery planning.
  • Practical checks and documentation to make recovery steps clearer under pressure.

Tools / Technologies

Nextcloud, rsync, Linux storage workflows, shared file permissions, backup routines, Backblaze, and documentation around recovery processes.

Security / Reliability Considerations

This write-up excludes folder structures, sensitive data handling details, internal paths, and backup configuration specifics. The important public takeaway is the approach: redundancy, permissions, restore thinking, and predictable routines matter more than a single tool choice.

What This Demonstrates to Employers

I can handle storage and backup work as an operational responsibility, not just a setup task. That includes thinking about permissions, user needs, failure cases, documentation, and the practical difference between “backed up” and “restorable.”

Skills Demonstrated
